# FitZone Club

### 💪 Сайт спортивного клуба – FitZone Club

📌 **Описание проекта**
FitZone Club — это веб-приложение для управления и взаимодействия с современным фитнес-клубом. Платформа позволяет посетителям просматривать актуальное расписание групповых и персональных тренировок, записываться на занятия, отслеживать свой прогресс и покупать абонементы онлайн. Для тренеров и администраторов предусмотрены инструменты управления расписанием и пользователями. Проект направлен на автоматизацию ключевых процессов клуба и создание удобного цифрового пространства для клиентов.

🎯 **Целевая аудитория:**
- Люди, ведущие активный образ жизни, от 16 лет
- Клиенты, желающие записываться на тренировки онлайн
- Пользователи, которые хотят следить за своим прогрессом
- Тренеры и администраторы фитнес-клуба

⚙ **Ключевой функционал**
- 📅 Просмотр расписания тренировок
- ✍️ Онлайн-запись на групповые и персональные занятия
- 👤 Личный кабинет с историей посещений и прогрессом
- 💳 Покупка абонементов онлайн
- ❤️ Добавление тренировок в избранное
- 🔐 Регистрация и авторизация пользователей (роли: user, trainer, admin)
- 🛠️ Админ-панель для управления расписанием, пользователями и абонементами

## 🛠 Технологический стек

### 🖥 Frontend

**Vue.js** – основной фреймворк.
*Почему выбран Vue.js:*
- **Компонентная архитектура** – позволяет разбить интерфейс на переиспользуемые компоненты (карточка тренировки, календарь, фильтры, шапка), что упрощает поддержку и масштабирование.
- **Низкий порог входа** – ускоряет разработку и упрощает поиск разработчиков.
- **Реактивность** – идеально подходит для динамического обновления расписания, фильтрации и корзины без перезагрузки страницы.
- **Производительность** – виртуальный DOM обеспечивает быструю работу даже при загрузке большого расписания.
- **Поддержка SPA** – создание одностраничного приложения с плавной навигацией.

**Vue Router** – маршрутизация.
*Почему выбран Vue Router:*
- Официальный роутер для Vue.js, полностью совместим с экосистемой.
- Поддержка динамических маршрутов (например, `/schedule/:id`).
- Возможность создания приватных маршрутов (`/profile`, `/admin`) с проверкой авторизации и роли.
- Удобная организация структуры SPA без перезагрузки страницы.

**Pinia** – государственное управление.
*Почему выбрана Pinia:*
- Официальное и современное решение для управления состоянием во Vue 3.
- Простота и типобезопасность (удобно работать с данными пользователя, расписанием, избранным).
- Легко разделять хранилища (userStore, scheduleStore, favoritesStore).

### 🛠 Backend

**Node.js** – среда выполнения.
*Почему выбран Node.js:*
- Единый язык (JavaScript) на фронтенде и бэкенде – общая экосистема и переиспользование кода.
- Высокая производительность при работе с асинхронными запросами (расписание, запись на тренировки, авторизация).
- Отлично подходит для создания REST API.
- Огромное количество библиотек и готовых решений.

**Express.js** – фреймворк для Node.js.
*Почему выбран Express.js:*
- Легкий и минималистичный, быстро настраивается.
- Удобная маршрутизация (например, `/api/schedule`, `/api/auth`, `/api/bookings`).
- Поддержка middleware для аутентификации (JWT), логирования и обработки ошибок.
- Простота создания REST API, что идеально для взаимодействия с фронтендом.

**PostgreSQL** – база данных.
*Почему выбрана PostgreSQL:*
- Реляционная структура идеально подходит для фитнес-клуба (таблицы: пользователи, тренировки, записи, абонементы).
- Поддержка сложных связей между таблицами (например, пользователь → записи → тренировки).
- Надежность, стабильность и соответствие стандартам SQL.
- Поддержка транзакций (важно при записи на тренировку, чтобы не допустить перезаписи).
- Широкое распространение и хорошая документация.

### 🌐 Инфраструктура

**Frontend хостинг:** GitHub Pages / Vercel
- Бесплатный хостинг, простая публикация из репозитория, автоматическое обновление.

**Backend хостинг:** Render.com / Railway
- Поддержка Node.js, автодеплой из GitHub, бесплатный тариф для старта.

**База данных:** Neon.tech / Supabase
- Облачный PostgreSQL с бесплатным тарифом, простое подключение и масштабирование.

## 🎨 Визуальное проектирование интерфейса (Вариант В: подробное текстовое описание)

|  [Логотип]    Главная | Расписание | Абонементы    [Аватар] Имя  |
|                                                       [Выйти]     |
+------------------------------------------------------------------+

### 2. Страница аутентификации (`/auth`)
**Тип доступа:** Публичный

**Основные блоки:**
1. **Header**: Логотип, ссылка «На главную».
2. **Форма входа**: Поле Email, поле Пароль, кнопка «Войти», ссылка на восстановление пароля.
3. **Форма регистрации**: Поле Имя, Email, Пароль, Кнопка «Зарегистрироваться».
4. **Форма восстановления пароля**: Поле Email, кнопка «Сбросить пароль» (отправка инструкции на почту).

**Динамический контент:**
- Регистрация: `POST /api/auth/register`
- Вход: `POST /api/auth/login`
- Отображение ошибок валидации (неверный email, пароль слишком короткий и т.д.).

### 3. Расписание (`/schedule`)
**Тип доступа:** Публичный (чтение)

**Основные блоки:**
1. **Header**.
2. **Фильтры**: По дате (календарь), по типу тренировки (йога, кроссфит, аэробика), по тренеру (выпадающий список).
3. **Сетка тренировок**: Карточка тренировки (время, название, тип, тренер, свободные места, кнопка «Записаться»).
4. **Состояния**:
   - *Загрузка*: Скелетоны карточек + спиннер.
   - *С данными*: Отображение сетки тренировок.
   - *Ошибка*: Сообщение «Не удалось загрузить расписание» и кнопка «Повторить».
5. **Footer**.

**Динамический контент:**
- Получение списка тренировок с фильтрацией: `GET /api/trainings?date=...&type=...&trainer=...`

### 4. Детальная страница тренировки (`/schedule/:id`)
**Тип доступа:** Публичный (чтение)

**Основные блоки:**
1. **Header**.
2. **Информация о тренировке**: Название, тип, дата и время, длительность, тренер (имя + фото), подробное описание, количество свободных мест.
3. **Блок записи**: Кнопка «Записаться» (активна, если есть места и пользователь авторизован), кнопка «В избранное».
4. **Блок с другими тренировками этого тренера** (карусель карточек).
5. **Footer**.

**Динамический контент:**
- Получение тренировки: `GET /api/trainings/:id`
- Похожие тренировки: `GET /api/trainings?trainerId=...&limit=3`

### 5. Личный кабинет (`/profile`)
**Тип доступа:** Приватный (авторизованный пользователь)

**Основные блоки:**
1. **Header**.
2. **Информация о пользователе**: Имя, Email, дата регистрации, активный абонемент (срок действия).
3. **Прогресс**: График посещений за месяц (заглушка или простой список).
4. **История записей**: Список предстоящих и прошедших тренировок с возможностью отменить запись.
5. **Кнопка «Редактировать профиль»** и кнопка «Выйти».
6. **Footer**.

**Динамический контент:**
- Данные пользователя: `GET /api/users/me`
- История записей: `GET /api/users/me/bookings`
- Отмена записи: `DELETE /api/bookings/:id`

### 6. Избранное (`/favorites`)
**Тип доступа:** Приватный

**Основные блоки:**
1. **Header**.
2. **Список избранных тренировок**: Карточки тренировок (как в расписании) с кнопкой «Удалить из избранного» и «Записаться».
3. **Сообщение**, если избранное пусто.
4. **Footer**.

**Динамический контент:**
- Получение избранных тренировок: `GET /api/users/me/favorites`
- Удаление: `DELETE /api/users/me/favorites/:trainingId`

### 7. Абонементы (`/memberships`)
**Тип доступа:** Приватный

**Основные блоки:**
1. **Header**.
2. **Список абонементов**: 3-4 карточки (название, срок действия, цена, список преимуществ, кнопка «Купить»).
3. **История покупок** (заглушка или простой список).
4. **Footer**.

**Динамический контент:**
- Получение абонементов: `GET /api/memberships`
- Покупка абонемента: `POST /api/memberships/buy`

### 8. Админ-панель (`/admin`)
**Тип доступа:** Приватный (только администраторы и тренеры)

**Основные блоки:**
1. **Header** с указанием роли Admin.
2. **Боковое меню**: Управление тренировками, управление пользователями, управление абонементами.
3. **Таблица тренировок** (для раздела управления тренировками):
   - Список всех тренировок (название, дата, тренер, свободные места)
   - Кнопки «Редактировать», «Удалить», «Добавить тренировку»
4. **Таблица пользователей** (для раздела управления пользователями):
   - Список пользователей (имя, email, роль)
   - Кнопка «Назначить тренером» / «Заблокировать»
5. **Footer**.

**Динамический контент:**
- CRUD операции над тренировками: `GET /api/admin/trainings`, `POST /api/admin/trainings`, `PUT /api/admin/trainings/:id`, `DELETE /api/admin/trainings/:id`
- Управление пользователями: `GET /api/admin/users`, `PUT /api/admin/users/:id/role`

## 🗄 Схема данных и API

### Основные модели данных

#### 1. Модель пользователя (User)

| Поле | Тип | Описание |
|---|---|---|
| `id` | integer | Уникальный идентификатор |
| `fullName` | string | Полное имя пользователя |
| `email` | string | Электронная почта (уникальная) |
| `passwordHash` | string | Хэш пароля |
| `role` | enum | Роль (`user`, `trainer`, `admin`) |
| `phone` | string | Номер телефона |
| `membershipId` | integer (FK) | ID активного абонемента (может быть null) |
| `createdAt` | datetime | Дата регистрации |

#### 2. Основная сущность — Тренировка (Training)

| Поле | Тип | Описание |
|---|---|---|
| `id` | integer | Уникальный идентификатор |
| `title` | string | Название тренировки |
| `type` | enum | Тип (`yoga`, `crossfit`, `aerobic`, `gym`) |
| `description` | text | Подробное описание |
| `trainerId` | integer (FK) | ID тренера (из User с role=trainer) |
| `dateTime` | datetime | Дата и время начала |
| `durationMinutes` | integer | Длительность в минутах |
| `maxSeats` | integer | Максимальное количество мест |
| `availableSeats` | integer | Доступное количество мест |
| `price` | decimal | Цена за одно занятие (без абонемента) |
| `isPopular` | boolean | Признак популярной тренировки |

#### 3. Запись на тренировку (Booking)

| Поле | Тип | Описание |
|---|---|---|
| `id` | integer | Уникальный идентификатор |
| `userId` | integer (FK) | ID пользователя |
| `trainingId` | integer (FK) | ID тренировки |
| `status` | enum | Статус (`confirmed`, `cancelled`, `completed`) |
| `bookedAt` | datetime | Дата и время записи |

#### 4. Абонемент (Membership)

| Поле | Тип | Описание |
|---|---|---|
| `id` | integer | Уникальный идентификатор |
| `name` | string | Название (например, «Безлимитный на месяц») |
| `price` | decimal | Цена |
| `durationDays` | integer | Срок действия в днях |
| `features` | json | Список преимуществ (например, `["Безлимит на групповые", "Гостевой пропуск"]`) |

### Связи между моделями

- **User** (1) ← → (M) **Booking**: Один пользователь может иметь много записей.
- **Training** (1) ← → (M) **Booking**: На одну тренировку может быть много записей.
- **User** (1) ← → (M) **Training** (как тренер): Один тренер может вести много тренировок.
- **User** (M) ← → (M) **Training** (избранное): Связь многие-ко-многим через таблицу `favorites`.

### API Эндпоинты

#### Аутентификация

- **Регистрация:** `POST /api/auth/register`
  - Тело: `{ "fullName": "...", "email": "...", "password": "...", "phone": "..." }`
  - Ответ: `{ "token": "jwt_token", "user": { ... } }`

- **Вход:** `POST /api/auth/login`
  - Тело: `{ "email": "...", "password": "..." }`
  - Ответ: `{ "token": "jwt_token", "user": { ... } }`

#### Тренировки (Training)

- **Получение списка тренировок:** `GET /api/trainings`
  - Query параметры: `?date=...&type=...&trainerId=...&popular=true`
  - Ответ: `[ { "id": 1, "title": "...", "trainer": { ... }, ... } ]`

- **Получение детальной информации о тренировке:** `GET /api/trainings/:id`
  - Ответ: `{ "id": 1, "title": "...", "description": "...", "trainer": { ... }, "availableSeats": 10, ... }`

#### Запись и избранное

- **Создать запись на тренировку:** `POST /api/bookings`
  - Тело: `{ "trainingId": 1 }`
  - Ответ: `{ "bookingId": 123, "status": "confirmed" }`

- **Отменить запись:** `DELETE /api/bookings/:id`

- **Добавить в избранное:** `POST /api/users/me/favorites/:trainingId`
- **Удалить из избранного:** `DELETE /api/users/me/favorites/:trainingId`

#### Абонементы

- **Получить список абонементов:** `GET /api/memberships`
  - Ответ: `[ { "id": 1, "name": "...", "price": 2999, "durationDays": 30 }, ... ]`

- **Купить абонемент:** `POST /api/memberships/buy`
  - Тело: `{ "membershipId": 1 }`
  - Ответ: `{ "message": "Абонемент активирован до 2025-06-01" }`

#### Админ-панель

- **Управление тренировками (CRUD):** `GET /api/admin/trainings`, `POST /api/admin/trainings`, `PUT /api/admin/trainings/:id`, `DELETE /api/admin/trainings/:id`
- **Управление пользователями:** `GET /api/admin/users`, `PUT /api/admin/users/:id/role`

## Роли и права доступа

| Роль | Возможности |
|---|---|
| **User** (Клиент) | Просмотр расписания, запись/отмена на тренировки, избранное, покупка абонементов, личный кабинет. |
| **Trainer** (Тренер) | Просмотр своего расписания, отметка посещений (в админ-панели), просмотр записанных на свои тренировки клиентов. |
| **Admin** (Администратор) | Полный CRUD над тренировками и пользователями, управление абонементами, назначение ролей. |
